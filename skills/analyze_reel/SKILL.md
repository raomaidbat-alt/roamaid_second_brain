# analyze_reel

## Что делает

Принимает ссылку на Instagram Reels или YouTube, отправляет на сервер анализа, получает транскрипцию и метаданные, затем через Gemini готовит структурированный разбор.

## Шаги выполнения

1. Извлечь URL из сообщения пользователя
2. Отправить POST на `http://150.241.116.28:8000/analyze` с телом `{"url": "<url>"}`
3. Получить из ответа: `content.transcript`, `meta.author`, `meta.view_count`, `meta.like_count`, `content.caption`
4. Если транскрипция недоступна — анализировать по описанию и метаданным
5. Сформировать промпт для Gemini и вернуть результат

## Промпт для Gemini

```
Ты эксперт по контент-маркетингу. Проанализируй видео и ответь строго в этом формате без markdown, звёздочек, решёток и заголовков:

О чём ролик: [1-2 предложения]

Что зашло:
- [пункт 1]
- [пункт 2]
- [пункт 3]

Улучшения:
- [пункт 1, 1-2 строки]
- [пункт 2, 1-2 строки]
- [пункт 3, 1-2 строки]

Идеи для новых роликов:
- [пункт 1]
- [пункт 2]
- [пункт 3]

Не больше 30 строк. Только текст.

Транскрипция: {transcript}
Автор: {author}, просмотры: {views}, лайки: {likes}, описание: {caption}
```

## Формат ответа

Эмодзи-секции:
- `Анализирую видео...` — пока обрабатывается
- `О чём ролик` — суть за 1-2 предложения
- `Что зашло` — 3 пункта со стрелками
- `Улучшения` — 3 конкретных пункта
- `Идеи для новых роликов` — 3 идеи

## Зависимости

- Сервер анализа: `http://150.241.116.28:8000/analyze`
- Переменная окружения: `GEMINI_API_KEY`
- Модель Gemini: `gemini-2.5-flash`

## Обработка ошибок

- Сервер недоступен → сообщить пользователю, предложить попробовать позже
- 429 от Gemini → ждать 30 сек, повторить (максимум 3 попытки, пауза 10 сек между запросами)
- Транскрипция отсутствует → анализировать по описанию и метаданным
