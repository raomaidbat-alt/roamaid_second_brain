# generate_thread

## Что делает

Генерирует 3 варианта треда с разными форматами крючков для одного аккаунта или ролика. Пользователь выбирает лучший вариант, бот запоминает что зашло.

## Шаги выполнения

1. Получить данные ролика/аккаунта (из предыдущего анализа или нового запроса к `/analyze`)
2. Сформировать промпт с тремя форматами крючков
3. Отправить в Gemini — получить 3 варианта треда
4. Показать пользователю крючок (первый пост) каждого варианта
5. Дать кнопки для выбора: Вариант 1, Вариант 2, Вариант 3
6. По выбору — отправить все посты треда
7. Сохранить выбор в feedback.json

## Промпт для Gemini

```
Ты копирайтер, эксперт по Instagram-маркетингу. Стиль: живой, конкретные цифры, без воды.

Данные: автор={author}, просмотры={views}, лайки={likes}, транскрипция={transcript}

Сгенерируй 3 варианта треда с разными крючками:

ПАРАДОКС: "{Описание блогера}, у которого есть {что есть}, но нет {чего нет}"
ЦИФРЫ: "При {X} подписчиках он зарабатывает только {Y} — хотя мог бы {Z}"
БОЛЬ: "{Имя}, {X} подписчиков, {конкретная боль которая мешает зарабатывать}"

Структура каждого треда (9 постов):
- Пост 1: крючок с реальными цифрами
- Посты 2-6: цифры воронки, что не работает и почему, анализ монетизации
- Посты 7-8: 3 конкретных шага что делать
- Пост 9: CTA "напиши слово РАЗБОР в комментариях, промпты забирай в закрепе t.me/+H1I_MHFuv603ZTdi"

Правила: каждый пост 3-5 предложений, конкретные цифры везде, без markdown, посты через ---

Формат вывода:
===ПАРАДОКС===
[посты через ---]
===ЦИФРЫ===
[посты через ---]
===БОЛЬ===
[посты через ---]
```

## Формат ответа пользователю

```
Генерирую 3 варианта треда...

[Вариант 1 — ПАРАДОКС]
{первый пост варианта}...

[Вариант 2 — ЦИФРЫ]
{первый пост варианта}...

[Вариант 3 — БОЛЬ]
{первый пост варианта}...

[Кнопки выбора: Вариант 1 | Вариант 2 | Вариант 3]
```

После выбора — отправить все 9 постов треда нумерованно `[1/9]`, `[2/9]` и т.д.

## Обучение на выборах

- Сохранять в `/root/feedback.json`: `{ts, url, hook_type, rating}`
- При следующей генерации читать статистику и подсказывать Gemini какой формат лучше заходил
- Если один формат получил 3+ оценки "good" — упоминать его в промпте

## Зависимости

- Переменная окружения: `GEMINI_API_KEY`
- Модель: `gemini-2.5-flash`
- Файл обратной связи: `/root/feedback.json`
- Пауза между Gemini-запросами: 10 сек, при 429 — 30 сек, максимум 3 попытки
