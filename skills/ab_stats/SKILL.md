# ab_stats

## Что делает

Читает данные из двух источников:
1. **Лист CRM** Google Sheets — статистика персональных сообщений А/Б/В
2. **Файл `/root/feedback.json`** — статистика тредов (крючки ПАРАДОКС/ЦИФРЫ/БОЛЬ)

Считает конверсию по каждому варианту и показывает сводный отчёт.

## Шаги выполнения

1. Открыть Google Sheets, лист `CRM`
2. Читать колонки: `Вариант (А/Б/В)`, `Ответил (да/нет)`
3. Считать для каждого варианта: отправлено (все строки) и ответили (где "да")
4. Читать `/root/feedback.json`
5. Считать по полю `hook_type`: сколько выбрано, сколько получили `rating: good`
6. Определить лучший вариант по конверсии (при минимум 3 наблюдениях)

## Алгоритм выбора лучшего варианта

```
конверсия = replied / sent * 100%
лучший = вариант с max конверсией при sent >= 3
```

## Формат ответа

```
Статистика сообщений (A/B тест):
Вариант А (цифры): отправлено 12, ответили 3 (25%)
Вариант Б (боль): отправлено 15, ответили 6 (40%)
Вариант В (комплимент): отправлено 8, ответили 1 (12%)
Лучший вариант: Б (боль)

Статистика тредов:
ПАРАДОКС: выбрано 5, понравилось 3 (60%)
ЦИФРЫ: выбрано 8, понравилось 6 (75%)
БОЛЬ: выбрано 4, понравилось 2 (50%)
Лучший крючок: ЦИФРЫ

Всего аудитов проведено: 35
Средняя конверсия сообщений: 28%
```

Если данных нет — сообщить что статистика ещё накапливается.

## Авто-обучение

После показа статистики — если есть явный лидер (конверсия выше других на 10%+ при 5+ наблюдениях):
- Напомнить что система уже автоматически использует этот вариант чаще (60% вероятность)
- Предложить сбросить статистику командой `/reset_stats` если нужно начать заново
